# Phuket Go Task Map

Статус: справочный файл для владельца проекта.

Важно: эта папка и этот файл не являются техническим заданием.

Нельзя использовать этот файл как источник требований для разработки.

Правильные источники требований находятся в `docs/`.

Этот файл нужен только для понимания, на каком крупном этапе находится проект.

## Легенда

- `[done]` - этап выполнен.
- `[in progress]` - этап сейчас в работе.
- `[todo]` - этап еще не начат.
- `[later]` - этап на будущее.

## Где мы сейчас

Сейчас проект находится на этапе:

`26. Финальная карта Google Sheets`

Код проекта еще не пишем.

## Общая цель

Путь проекта:

1. `Phuket Go`
2. `Bangkok Go`
3. `Pattaya Go`
4. `Thailand Go`
5. `Vietnam Go`
6. `Asia Go`
7. `World Go`

Идея: сначала доказать модель на Пхукете, потом масштабировать ее на другие города, страны, регионы и мировой уровень.

Важное правило: задел на масштабирование учитываем сразу.

Даже когда делаем MVP только для Пхукета, не принимаем решения, которые мешают будущим городам, странам, языкам, валютам, партнерам и категориям услуг.

## Следующий крупный результат

Нужно получить документ с финальной картой Google Sheets:

- точные колонки;
- обязательные поля;
- статусы;
- кто какие поля обновляет;
- как считать комиссию;
- как потом переносить данные в Postgres.

## Как обновлять этот файл

Когда крупный этап завершен:

1. Меняем его статус на `[done]`.
2. Следующий этап меняем на `[in progress]`.
3. Не добавляем слишком мелкие внутренние задачи.
4. Не используем этот файл как источник требований.

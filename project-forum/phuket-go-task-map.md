## Где мы сейчас

Сейчас проект находится на этапе локальной разработки MVP:

`76. Публичный сайт` + финальная локальная проверка заявки, Telegram и Google Sheets.

Код проекта уже пишется. Подготовительный блок 39-73 закрыт на уровне документов и локальной реализации, но этот файл остается только трекером прогресса.

Главное текущее правило: ничего не деплоим, не загружаем в облако и не начинаем публикацию, пока сайт и основной путь заявки не будут локально доведены до 10/10.

Важно по статусам: `[done]` в таблицах ставим только для этапов, которые уже реализованы, проверены и зафиксированы в git. Локальные незакоммиченные правки записываем как текущий хвост, но не считаем финально закрытыми.

## Текущее состояние локальной разработки

Уже сделано и зафиксировано:

- собран рабочий monorepo `website`/`webapp`/`backend`/`packages/contracts`;
- публичный сайт читает 54 экскурсии, статьи, FAQ, контакты и условия;
- форма заявки создает лид без оплаты и без обязательного логина;
- backend сохраняет заявки в Postgres, пишет Google Sheets и отправляет Telegram-уведомления;
- партнерская Telegram-клавиатура отполирована: сначала одна главная кнопка `Принять заявку`, после принятия появляются рабочие действия;
- callback-data `accept`, `paid`, `problem`, `decline` сохранены совместимыми;
- админская очередь заявок умеет фильтры, CSV, bulk status, заметки, ручной Sheets sync и смену статуса;
- пройдены unit/integration/E2E проверки области заявок, Telegram, Google Sheets, админки и базовых build/typecheck;
- внешний Telegram smoke и внешний Google Sheets smoke проходили локально без коммита секретов;
- добавлена поддержка build-time base path для будущего хостинга, но релиз не начинаем.

Локально сделано в текущем рабочем дереве и ждет отдельного commit/push:

- `website` синхронизирует 569 медиафайлов из `docs/03-service-catalog/media/excursions/*/final/carousel` в `website/public/media`;
- публичный build больше не зависит от localhost media URL;
- главная получила реальный фото-hero, страницы экскурсий получили более аккуратную мобильную галерею и header;
- `website` и общий build/typecheck после этих правок проходили локально.

Что еще нужно сделать до локального 10/10:

1. Довести верстку публичного сайта: главная, каталог, карточки, страницы экскурсий, статьи, FAQ, контакты, условия, мобильные и desktop состояния.
2. Сделать визуальный аудит всех ключевых публичных страниц: нет наложений текста, сломанных сеток, пустых фото, localhost-ссылок и слабых первых экранов.
3. Провести финальную ручную чистку фото перед публикацией: слабые/повторные/неподходящие фото, лица людей, соответствие карточке, статус `approved` там, где фото точно идет в релиз.
4. Повторить полный локальный smoke: сайт -> заявка -> backend -> Telegram admin/partner -> Google Sheets -> callback `accept`/`paid`/`problem`/`decline`.
5. Довести или явно отложить пользовательский Telegram-бот, личный кабинет, баллы и неполные разделы админки.
6. Обновить docs после финального состояния сайта и только потом возвращаться к deployment docs.

## Общая цель

Путь проекта:

1. `Phuket Go`
2. `Bangkok Go`
3. `Pattaya Go`
4. `Thailand Go`
5. `Vietnam Go`
6. `Asia Go`
7. `World Go`

Идея: сначала доказать модель на Пхукете, потом масштабировать ее на другие города, страны, регионы и мировой уровень.

Важное правило: задел на масштабирование учитываем сразу.

Даже когда делаем MVP только для Пхукета, не принимаем решения, которые мешают будущим городам, странам, языкам, валютам, партнерам и категориям услуг.

## Следующий крупный результат

Нужно довести локальный MVP до 10/10 перед публикацией:

- публичный сайт должен выглядеть и работать уверенно на desktop и mobile;
- полный путь заявки должен проходить локально через сайт, backend, Telegram, Google Sheets и админку;
- незавершенные MVP-поверхности нужно либо закрыть, либо явно отложить из первого релиза;
- до этого не начинаем деплой, загрузки в облако, настройку production ресурсов и публикацию.

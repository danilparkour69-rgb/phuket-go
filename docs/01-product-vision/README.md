# Этап 1. Product Vision

## Цель

Сформулировать видение продукта: что строим, для кого, какую проблему решаем, чем отличаемся и почему пользователь должен выбрать именно наш сервис.

## Предварительное описание продукта

Туристическая SaaS-экосистема для Пхукета, которая начинается как маркетплейс услуг и постепенно развивается в super app: экскурсии, транспорт, недвижимость, документы, финансы, бытовые услуги, полезная информация, геймификация и партнерская сеть.

- [x] Описать целевую аудиторию: туристы, экспаты, долгосрочные резиденты, инвесторы, партнеры.
- [x] Сформулировать главную проблему каждой аудитории.
- [x] Сформулировать ценностное предложение продукта.
- [x] Определить отличие от обычного сайта экскурсий, Telegram-каталога и агрегаторов.
- [x] Описать продуктовую миссию на 1-2 абзаца.
- [x] Определить границы первой версии: что входит, что не входит.
- [x] Подготовить короткий elevator pitch.

## Итог этапа

Документ `product-vision.md`, в котором есть:

- описание продукта;
- целевые аудитории;
- ключевые боли;
- ценностное предложение;
- конкурентное отличие;
- границы MVP;
- формулировка миссии.

Дополнительно:

- `product-ideas.md` - входящие продуктовые идеи и позиционирование к будущему обсуждению. Это не очередь разработки.

# Account Screens Map

Статус: [x] done

Кодинг проекта пока не начинаем.

Документ описывает личный кабинет пользователя Phuket Go для MVP и задел под будущие города, страны и приложения.

## Главное решение

Личный кабинет относится к `webapp` из шаблона `vibe`.

Публичные SEO-страницы экскурсий и статей остаются в `website`.

Пользователь может оставить заявку без регистрации. Регистрация нужна для истории заявок, баллов, уровня и будущей игры.

## Кто может пользоваться

| Тип пользователя | Что доступно |
| --- | --- |
| Гость | Смотреть экскурсии, читать статьи, отправлять заявку |
| Авторизованный пользователь | Все действия гостя, история заявок, баллы, уровень, профиль |

В MVP не делаем:

- онлайн-оплату;
- сложные настройки аккаунта;
- чат внутри кабинета;
- полноценную игру;
- кабинет партнера внутри личного кабинета пользователя.

## Способы входа

В MVP используем бесплатные способы входа:

- Telegram;
- Google;
- email.

Телефон через SMS не используем в MVP, потому что SMS может быть платным.

ВКонтакте и другие способы входа можно добавить позже.

## Экран 1. Вход

Задача экрана: дать пользователю быстро войти и не мешать гостю оставить заявку.

Что показываем:

- кнопка входа через Telegram;
- кнопка входа через Google;
- вход по email;
- объяснение, что заявка доступна и без регистрации.

Что происходит после входа:

- создается профиль пользователя;
- пользователю начисляется 100 баллов;
- пользователь получает первый уровень;
- если у пользователя уже были заявки с тем же телефоном или Telegram, в будущем можно связать их с аккаунтом вручную или через подтверждение.

## Экран 2. Профиль

Задача экрана: хранить базовые данные для быстрых заявок.

Поля:

| Поле | Для чего нужно |
| --- | --- |
| Имя | Подставлять в форму заявки |
| Телефон | Связь с пользователем |
| Telegram | Связь и будущий Telegram-бот пользователя |
| Email | Вход и резервная связь |
| Способ входа | Понимать, через что создан аккаунт |
| Город по умолчанию | Сейчас Phuket, позже Bangkok, Pattaya и другие |
| Язык | В MVP русский, позже английский |

Пользователь может изменить:

- имя;
- телефон;
- Telegram.

Пользователь не может сам менять:

- баллы;
- уровень;
- историю заявок;
- статусы заявок.

## Экран 3. Мои заявки

Задача экрана: показать пользователю, что он уже отправлял и на какие даты.

В списке заявок показываем:

| Поле | Пример |
| --- | --- |
| Название экскурсии | Пхи-Пхи |
| Дата | 12 августа |
| Количество людей | 2 взрослых |
| Статус | Взята в работу |
| Город | Phuket |
| Дата создания заявки | 28 июня 2026 |

Пользователь видит только свои заявки.

Партнер в пользовательском кабинете не показывается.

## Экран 4. Детали заявки

Задача экрана: показать понятное состояние одной заявки.

Показываем:

- название экскурсии;
- фото или обложку экскурсии;
- дату;
- количество людей;
- имя пользователя;
- телефон;
- Telegram, если указан;
- комментарий пользователя;
- текущий статус;
- историю статусов;
- что будет дальше.

Статусы для пользователя:

| Внутренний статус | Текст для пользователя |
| --- | --- |
| `new` | Заявка получена |
| `waiting_partner` | Ожидаем подтверждение |
| `accepted` | Заявка взята в работу |
| `declined` | Заявка отклонена |
| `completed` | Экскурсия состоялась |
| `cancelled` | Заявка отменена |

В MVP пользователь не редактирует заявку после отправки.

Если нужно что-то изменить, пользователь связывается с Phuket Go или менеджером через Telegram/телефон.

## Экран 5. Баллы и уровень

Задача экрана: показать базовую геймификацию без полноценной игры.

В MVP показываем:

- уровень пользователя;
- общее количество баллов;
- баллы за регистрацию;
- баллы за завершенные экскурсии;
- баллы за отзывы, если отзыв одобрен админом.

Старт:

- регистрация = 100 баллов;
- пользователь сразу получает первый уровень.

Баллы пока нужны только для уровня.

Награды, подарки, топы месяца и геолокационная игра относятся к будущим версиям.

## Экран 6. Отзывы пользователя

Задача экрана: подготовить базу для отзывов, но не усложнять MVP.

В MVP отзыв публикует админ вручную.

Пользователь может оставить отзыв только по экскурсии, которая состоялась.

Отзывы с фото в будущем дают больше баллов.

Звезды в MVP не используем.

## Будущий экран. Хочу посетить

Задача экрана: сохранить интерес пользователя до того, как он готов выбрать дату или оставить заявку.

Пользователь может нажать сердечко на карточке экскурсии, места или будущей услуги.

В списке показываем:

| Поле | Для чего нужно |
| --- | --- |
| Название | Понять, что пользователь сохранил |
| Тип | Экскурсия, место, маршрут или услуга |
| Фото | Быстро вспомнить желание |
| Город | Phuket, позже Bangkok, Pattaya и другие |
| Дата сохранения | Понимать свежесть интереса |
| Кнопка действия | Оставить заявку, открыть карточку или удалить из списка |

Смысл: это мягкий шаг между "мне интересно" и "я готов планировать дату".

В MVP этот экран можно не делать, но структуру личного кабинета стоит держать готовой к избранному.

## Экран 7. Связь с Telegram

Задача экрана: связать сайт, личный кабинет и пользовательский Telegram-бот.

Показываем:

- Telegram подключен или нет;
- кнопку подключения Telegram;
- ссылку на пользовательского Telegram-бота;
- короткий текст, что в боте можно смотреть статус заявки.

Пользовательский Telegram-бот должен показывать те же заявки и статусы, что и личный кабинет.

## Масштабирование

Личный кабинет сразу проектируем так, чтобы позже он работал не только для Phuket Go.

Будущие уровни:

- Phuket Go;
- Bangkok Go;
- Pattaya Go;
- Thailand Go;
- Vietnam Go;
- Asia Go;
- World Go.

Что нужно заложить:

- у заявки должен быть город;
- у заявки должна быть страна;
- у услуги должна быть категория;
- у пользователя должна быть единая история по всем городам;
- баллы пользователя должны работать на уровне всей платформы;
- язык и валюта должны расширяться без переделки кабинета.

## Безопасность

Правила:

- пользователь видит только свои заявки;
- пользователь не видит внутренние заметки админа;
- пользователь не видит комиссию партнера;
- пользователь не видит долг партнера;
- пользователь не может менять статус заявки;
- пользователь не может менять баллы вручную.

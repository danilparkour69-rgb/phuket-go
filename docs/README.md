# План создания SaaS туристического маркетплейса

Дата создания: 2026-06-27

Этот каталог хранит рабочую документацию проекта: видение продукта, требования, исследование, архитектурные решения, MVP, порядок разработки и актуальные решения для локальной реализации.

## Как пользоваться документами

- Главный список этапов находится в [00-project-control/master-plan.md](00-project-control/master-plan.md).
- В каждой папке этапа есть свой `README.md` с итогом этапа, подзадачами и критериями готовности.
- Когда задача выполнена, меняем `[ ]` на `[x]` и добавляем дату/краткий комментарий.
- Если по подзадаче появляются отдельные материалы, сохраняем их в папку соответствующего этапа.
- Код проекта уже начат локально; документацию обновляем по мере уточнения продуктовых и технических решений.

## Статусы

- `[ ]` - не начато
- `[~]` - в работе
- `[x]` - done
- `[!]` - заблокировано, нужен ответ или решение

## Структура

- `00-project-control` - мастер-план, правила ведения документации, журнал решений.
- `01-product-vision` - видение продукта и позиционирование.
- `02-market-research` - конкуренты и рынок.
- `03-service-catalog` - каталог услуг и логика карточек.
- `04-users-and-roles` - роли пользователей и личные кабинеты.
- `05-lead-processing` - заявки, статусы, маршрутизация партнерам.
- `06-monetization` - бизнес-модель и источники дохода.
- `07-seo-and-content` - SEO-структура, страницы, контент.
- `08-gamification` - баллы, уровни, достижения, квесты.
- `09-platforms-and-architecture` - платформы, архитектура, API, база данных.
- `10-mvp-roadmap` - MVP, этапы запуска, приоритеты.
- `11-open-questions` - вопросы, без которых нельзя переходить к разработке.

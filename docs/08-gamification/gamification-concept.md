# Gamification Concept

Статус: [~] draft

## Роль геймификации в продукте

В MVP геймификация минимальная. Цель - заложить базу для будущей игры, не разрабатывая полноценную игровую механику.

## MVP-механики

- пользователь регистрируется;
- пользователь получает 100 стартовых баллов;
- зарегистрированный пользователь получает первый уровень.
- пользователь получает баллы за заказанную экскурсию;
- авторизованный пользователь видит в карточке или на странице экскурсии, сколько баллов получит после посещения;
- более дорогие экскурсии дают больше баллов;
- отзыв дает баллы, но меньше, чем дорогая экскурсия;
- отзыв с фото дает больше баллов, чем обычный текстовый отзыв;
- приглашение друга дает больше баллов, чем отзыв и регистрация, но меньше, чем экскурсия.

## Будущие механики

- уровни;
- опыт;
- достижения;
- задания;
- реферальная система;
- награды;
- посещение локаций;
- туристические квесты.

## Логика сложности

С каждым новым уровнем получать следующий уровень становится сложнее.

То есть для второго уровня нужно меньше баллов, чем для третьего, для третьего меньше, чем для четвертого, и так далее.

## Приоритет баллов

От меньшего к большему:

1. Регистрация.
2. Отзыв.
3. Отзыв с фото.
4. Приглашенный друг.
5. Экскурсия.
6. Дорогая экскурсия.

## Использование баллов

В MVP и базовой логике баллы нужны только для уровней.

Тратить баллы нельзя.

## Отображение баллов на сайте

Гость:

- видит экскурсии;
- читает статьи;
- оставляет заявку без регистрации;
- не видит персональный блок баллов.

Авторизованный пользователь:

- видит, сколько баллов получит после посещения тура или места;
- видит свой уровень;
- видит прогресс до следующего уровня, если это не перегружает карточку;
- после подтвержденного посещения получает баллы в историю профиля.

## Награды

Награды и подарки появятся в будущем, когда проект выйдет на хорошую монетизацию.

Идея будущего: топы месяца получают награды.

## Защита от накрутки

- отзыв можно оставить только после посещения экскурсии;
- посещение мест подтверждается геометкой.

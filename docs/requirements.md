# Phuket Go Requirements

Статус: рабочая продуктовая база для разработки MVP

## 1. Продукт

Phuket Go - русскоязычный travel marketplace MVP по экскурсиям на Пхукете.

Главная задача первой версии:

- помочь туристу быстро выбрать экскурсию;
- дать понятную SEO-страницу по каждой экскурсии;
- принять заявку без онлайн-оплаты;
- передать заявку владельцу проекта и исполнителю через Telegram;
- сохранить основу для личного кабинета, истории заявок и баллов.

## 2. Аудитория и география MVP

- аудитория: русскоязычные туристы;
- география MVP: только Пхукет;
- категория MVP: только экскурсии;
- основная валюта партнера и базы: THB;
- публично на сайте: цена в батах крупно, рубли как ориентир мелко.

## 3. Архитектурная раскладка

- `website` - публичный SEO-сайт, главная, каталог, страницы экскурсий, статьи, FAQ, контакты, условия;
- `webapp` - закрытая часть после входа, личный кабинет пользователя и админка;
- `backend` - API, заявки, авторизация, Telegram-боты, Google Sheets, бизнес-логика;
- `packages/contracts` - общие схемы данных и API-контракты.

Если ранние идеи Phuket Go конфликтуют с архитектурой шаблона `vibe`, реализацию подстраиваем под `vibe`, не ломая продуктовую логику.

## 4. Публичный сайт MVP

На публичном сайте должны быть:

- главная страница с обещанием `Пхукет доступен в одно касание`;
- каталог экскурсий;
- отдельная SEO-страница каждой экскурсии;
- статьи;
- FAQ;
- контакты;
- условия бронирования.

### Главная

Главная должна показывать:

- миссию и ценность сервиса;
- популярные экскурсии;
- категории;
- блок доверия;
- как проходит бронирование;
- статьи и FAQ.

### Каталог

Каталог должен показывать:

- карточки экскурсий;
- фильтр по категории;
- фильтр по цене;
- цену в THB;
- ориентир в RUB;
- короткий эмоциональный текст;
- CTA на заявку.

Партнера в карточках публично не показываем.

### Страница экскурсии

Страница экскурсии должна содержать:

- название;
- фотогалерею;
- цену в THB и ориентир в RUB;
- длительность;
- эмоциональное описание;
- маршрут и программу;
- что включено;
- что оплачивается отдельно;
- что взять с собой;
- ограничения;
- страховку;
- отзывы;
- FAQ по экскурсии;
- похожие экскурсии;
- связанные статьи;
- повторяющиеся CTA на бронирование.

Правило текста:

Сначала продаем мечту, воспоминание и ощущения, затем даем факты, условия и заявку.

### Статьи

Статьи нужны для SEO и входа в каталог.

Структура:

- `/articles`
- `/articles/{slug}`

Каждая статья должна:

- отвечать на конкретный вопрос пользователя;
- подводить к связанной экскурсии или действию;
- вести в каталог или на карточку экскурсии.

## 5. Заявка

Онлайн-оплаты в MVP нет.

Обязательные поля:

- имя;
- телефон.

Дополнительные поля:

- Telegram;
- дата;
- количество людей;
- комментарий.

После отправки:

- заявка сохраняется;
- заявка уходит владельцу проекта в Telegram;
- заявка уходит исполнителю в Telegram;
- пользователь видит подтверждение;
- авторизованный пользователь видит заявку в истории.

После отправки пользователь выбирает удобный канал связи:

- Telegram;
- WhatsApp;
- Max.

Выбранный канал сохраняется в заявке.

## 6. Статусы заявки

Минимальные статусы MVP:

- `new`;
- `waiting_partner`;
- `accepted`;
- `declined`;
- `completed`;
- `cancelled`.

История статусов должна хранить:

- кто сменил статус;
- когда сменил статус;
- старый статус;
- новый статус.

## 7. Telegram

### Бот исполнителя

Исполнитель получает в Telegram:

- номер или идентификатор заявки;
- экскурсию;
- имя клиента;
- телефон;
- Telegram, если есть;
- дату;
- количество людей;
- комментарий.

Базовые действия исполнителя:

- `Взята в работу`;
- `Отклонено`;
- `Оказана`.

### Бот пользователя

Пользователь через Telegram должен иметь возможность:

- видеть свои заявки;
- видеть текущий статус;
- получать уведомления при смене статуса.

## 8. Авторизация и личный кабинет

Гость может оставить заявку без регистрации.

Способы входа MVP:

- Telegram;
- Google;
- email.

Телефонная SMS-авторизация в MVP не входит.

Личный кабинет должен показывать:

- профиль;
- историю заявок;
- текущие статусы;
- баллы;
- уровень.

При регистрации пользователь получает:

- 100 стартовых баллов;
- первый уровень.

## 9. Админка MVP

Админка должна позволять владельцу проекта управлять:

- экскурсиями;
- заявками;
- пользователями;
- партнерами;
- отзывами;
- статьями;
- комиссиями;
- аналитикой.

## 10. Партнеры

В MVP партнер работает без личного кабинета, через Telegram.

Партнер:

- не показывается публично в карточке;
- получает заявки в Telegram;
- позже сможет скрывать карточки, но не удалять их.

Дайвинг маршрутизируется отдельному исполнителю.

## 11. Фото и контент

Для MVP используем локально сохраненную медиабазу по направлениям.

Источник для разработки:

- папки направлений в `docs/03-service-catalog/media/excursions/`;
- `final/carousel/`;
- `media-manifest.json`.

Перед публикацией потребуется финальная ручная проверка фото.

## 12. Ограничения MVP

В MVP не делаем:

- онлайн-оплату;
- полноценный кабинет партнера;
- мобильное приложение;
- Telegram Mini App;
- англоязычную версию;
- недвижимость, визы, документы, аренду байков, обмен валют;
- удаление карточек партнером.

## 13. Масштабирование

Структура данных должна позволять путь:

1. Phuket Go
2. Bangkok Go
3. Pattaya Go
4. Thailand Go
5. Vietnam Go
6. Asia Go
7. World Go

Поэтому решения не должны жестко прибивать проект навсегда к одному городу и одной категории.

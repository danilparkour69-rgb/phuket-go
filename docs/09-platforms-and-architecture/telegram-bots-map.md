# Telegram Bots Map

Статус: [x] done

Документ описывает карту Telegram-ботов Phuket Go MVP.

## Главное правило

В MVP нужны два Telegram-бота:

- бот для партнеров;
- бот для пользователей.

Backend живет в `backend`.

Общие схемы кнопок, статусов и сообщений позже должны быть в `packages/contracts`.

Тексты сообщений и кнопок для MVP зафиксированы здесь:

- `../10-mvp-roadmap/interface-microcopy-mvp.md`

## Зачем нужны боты

### Бот партнера

Партнер получает заявку и меняет статус без входа в админку.

### Бот пользователя

Пользователь видит статус заявки и получает уведомления.

## Роли

| Роль | Что делает в Telegram |
| --- | --- |
| Админ | Получает все новые заявки и ошибки |
| Партнер | Получает свои заявки и меняет статус |
| Пользователь | Получает подтверждения и смотрит статус |
| Система | Отправляет уведомления и пишет историю |

## Статусы заявки

| Статус | Отображение для пользователя | Кто ставит |
| --- | --- | --- |
| `new` | Заявка отправлена | Система |
| `waiting_partner` | Ожидаем подтверждения | Система или админ |
| `accepted` | Заявка взята в работу | Партнер или админ |
| `declined` | Заявка отклонена | Партнер или админ |
| `completed` | Экскурсия оказана | Партнер или админ |
| `cancelled` | Заявка отменена | Админ |

## Бот партнера

### Цель

Партнер должен быстро понять заявку и нажать нужную кнопку.

### Главное меню партнера

Кнопки:

- `Новые заявки`;
- `В работе`;
- `Оказанные`;
- `Помощь`.

В MVP можно начать без сложного меню, если все действия приходят прямо в сообщении заявки.

### Сообщение новой заявки партнеру

Текст:

```text
Новая заявка Phuket Go

Заявка: #{public_number}
Экскурсия: {excursion_title}
Дата: {preferred_date}
Количество людей: {people_count}

Клиент: {customer_name}
Телефон: {customer_phone}
Telegram: {customer_telegram}

Комментарий: {customer_comment}
```

Кнопки:

- `Взять в работу`;
- `Отклонить`;
- `Связаться с клиентом`.

### Когда партнер нажал `Взять в работу`

Что происходит:

1. Статус заявки меняется на `accepted`.
2. Заполняется `accepted_at`.
3. В Google Sheets обновляется строка.
4. Админ получает уведомление.
5. Пользователь получает уведомление, если он связан с ботом.

Сообщение партнеру:

```text
Заявка #{public_number} взята в работу.
Свяжитесь с клиентом и подтвердите детали.
```

Кнопки после принятия:

- `Оказана`;
- `Проблема`;

### Когда партнер нажал `Отклонить`

Что происходит:

1. Статус заявки меняется на `declined`.
2. Заполняется `declined_at`.
3. Админ получает уведомление.
4. Пользователь получает уведомление.

Сообщение партнеру:

```text
Заявка #{public_number} отклонена.
Администратор увидит это и решит, что делать дальше.
```

### Когда партнер нажал `Оказана`

Что происходит:

1. Статус заявки меняется на `completed`.
2. Заполняется `completed_at`.
3. Начисляется комиссия.
4. Админ получает уведомление.
5. Пользователь получает уведомление.

Сообщение партнеру:

```text
Заявка #{public_number} отмечена как оказанная.
Комиссия будет учтена в месячном расчете.
```

### Кнопка `Проблема`

Нужна, если партнер не может обработать заявку нормально.

Варианты:

- `Клиент не отвечает`;
- `Нет мест`;
- `Нужна помощь админа`;
- `Другая причина`.

После выбора причины админ получает уведомление.

## Бот пользователя

### Цель

Пользователь должен понимать, что происходит с заявкой.

### Главное меню пользователя

Кнопки:

- `Мои заявки`;
- `Связаться с Phuket Go`;
- `Каталог экскурсий`;
- `Помощь`.

### Сообщение после отправки заявки

Текст:

```text
Заявка отправлена.

Номер заявки: #{public_number}
Экскурсия: {excursion_title}

Мы передали заявку менеджеру и партнеру.
Когда статус изменится, вы получите уведомление.
```

Если пользователь не связан с Telegram-ботом, это сообщение показывается на сайте, а связь идет по телефону или Telegram, который он указал в форме.

### Статус `waiting_partner`

Текст:

```text
Ожидаем подтверждения.

Заявка #{public_number} передана партнеру.
Мы сообщим, когда появится ответ.
```

### Статус `accepted`

Текст:

```text
Заявка взята в работу.

Партнер свяжется с вами, чтобы подтвердить детали.
```

### Статус `declined`

Текст:

```text
Заявка отклонена партнером.

Администратор Phuket Go проверит ситуацию и при необходимости предложит другой вариант.
```

### Статус `completed`

Текст:

```text
Экскурсия отмечена как оказанная.

Спасибо, что выбрали Phuket Go.
```

Позже здесь можно добавить предложение оставить отзыв.

### Статус `cancelled`

Текст:

```text
Заявка отменена.

Если нужна помощь с выбором другой экскурсии, свяжитесь с Phuket Go.
```

## Бот администратора

Отдельный бот админа не обязателен.

В MVP можно использовать тот же backend-канал уведомлений в Telegram.

Админ получает:

- все новые заявки;
- ошибки маршрутизации;
- заявки без партнера;
- отклоненные заявки;
- заявки с проблемой;
- оказанные заявки;
- уведомления о комиссии;
- ошибки отправки уведомлений.

## Сообщение новой заявки админу

```text
Новая заявка Phuket Go

Заявка: #{public_number}
Статус: new
Экскурсия: {excursion_title}
Партнер: {partner_name}

Клиент: {customer_name}
Телефон: {customer_phone}
Telegram: {customer_telegram}
Дата: {preferred_date}
Людей: {people_count}

Комментарий: {customer_comment}
```

Кнопки для админа можно добавить позже.

В MVP достаточно уведомления и ссылки на админку.

## Ошибки и исключения

### Нет партнера

Админу:

```text
Ошибка маршрутизации.

Заявка #{public_number} создана, но партнер не назначен.
Проверьте заявку в админке.
```

### Не удалось отправить партнеру

Админу:

```text
Не удалось отправить заявку партнеру.

Заявка: #{public_number}
Партнер: {partner_name}
Проверьте Telegram партнера.
```

### Не удалось уведомить пользователя

Админу:

```text
Не удалось уведомить пользователя.

Заявка: #{public_number}
Клиент: {customer_name}
Связь возможна по телефону или Telegram из заявки.
```

### Партнер нажал кнопку повторно

Партнеру:

```text
Статус заявки уже изменен.
Актуальный статус: {status}
```

### Неверный партнер

Если партнер пытается открыть не свою заявку:

```text
Эта заявка не относится к вашему аккаунту.
Если это ошибка, напишите администратору Phuket Go.
```

## Какие данные нужны Telegram-ботам

### Для партнера

- `lead_id`;
- `public_number`;
- `excursion_title_snapshot`;
- `preferred_date`;
- `people_count`;
- `customer_name`;
- `customer_phone`;
- `customer_telegram`;
- `customer_comment`;
- `partner_id`;
- `partner_telegram`;
- `status`.

### Для пользователя

- `user_id`, если авторизован;
- `telegram_chat_id`, если связан;
- `public_number`;
- `excursion_title_snapshot`;
- `status`;
- `preferred_date`;
- `people_count`.

### Для админа

- все данные заявки;
- ошибки маршрутизации;
- ошибки отправки;
- изменения статусов;
- комиссии.

## Что пишем в историю

Каждое действие партнера или админа должно создать запись:

- заявка;
- старый статус;
- новый статус;
- кто изменил;
- когда изменил;
- комментарий или причина.

## Что обновляем в Google Sheets

При смене статуса обновляем:

- `status`;
- `updated_at`;
- `accepted_at`, если статус `accepted`;
- `declined_at`, если статус `declined`;
- `completed_at`, если статус `completed`;
- `cancelled_at`, если статус `cancelled`;
- `last_status_changed_by_type`;
- `last_status_changed_by_id`;
- `last_status_changed_at`;
- `partner_note`, если есть.

При статусе `completed` дополнительно:

- `commission_total_thb`;
- `commission_status`;
- `commission_period_month`.

## Что не делаем в MVP ботов

- оплату через Telegram;
- сложные меню;
- рассылки;
- массовые уведомления;
- партнерский кабинет внутри Telegram;
- Telegram Mini App;
- отзывы через бот;
- начисление подарков через бот.

## Telegram polish перед публикацией

Это не блокер текущей локальной разработки, а небольшой блок доработок перед публикацией на сервер:

- подготовить или подключить Telegram-стикеры для более живых системных сообщений;
- привести кнопки, emoji и короткие подписи к более приятному и спокойному стилю;
- проверить последовательность уведомлений, чтобы она воспринималась логично и не перегружала партнера;
- рассмотреть сценарий, где у новой заявки сначала есть одно главное действие `Принять заявку`, а отклонение доступно только после принятия/контакта с клиентом, если это не мешает реальному операционному процессу;
- финально сверить этот сценарий с админкой, статусами заявки и Google Sheets, чтобы не появилось расхождений между каналами.

## Задел на масштабирование

Боты должны учитывать будущие:

- города;
- страны;
- языки;
- валюты;
- разные категории услуг;
- разные партнерские команды.

В MVP значения по умолчанию:

- город: Phuket;
- страна: Thailand;
- язык: ru;
- категория: excursion.

- уведомления админу;
- кнопки;
- сообщения;
- статусы;
- ошибки;
- обновление Google Sheets;
- история статусов;
- что не делаем в MVP;
- задел на масштабирование.

# Admin Screens Map

Статус: [x] done

Документ описывает карту экранов админки Phuket Go MVP.

## Главное правило

Админка живет в `webapp` шаблона `vibe`.

Backend/API живет в `backend`.

Общие схемы данных должны быть в `packages/contracts`.

MVP админки делаем для одного администратора: владельца проекта.

## Главная задача админки

Админка должна позволить владельцу Phuket Go управлять MVP без программиста:

- видеть заявки;
- управлять экскурсиями;
- видеть пользователей;
- управлять отзывами;
- управлять статьями;
- видеть партнеров;
- видеть комиссии;
- видеть базовую аналитику.

## 1. Дашборд

### Цель

Админ быстро видит, что происходит сегодня.

### Показываем

- новые заявки;
- заявки в ожидании партнера;
- заявки в работе;
- оказанные заявки;
- отклоненные заявки;
- сумма начисленной комиссии;
- сумма долгов;
- топ экскурсий по заявкам;
- партнеры, которые не ответили.

### Действия

- открыть список заявок;
- открыть заявку;
- открыть аналитику;
- открыть комиссии.

## 2. Заявки

### Цель

Главный рабочий экран админа.

### Список заявок

Поля в таблице:

- номер заявки;
- дата создания;
- статус;
- экскурсия;
- дата экскурсии;
- клиент;
- телефон;
- Telegram;
- количество людей;
- партнер;
- комиссия;
- последний ответ партнера;
- заметка админа.

### Фильтры

- статус;
- дата создания;
- дата экскурсии;
- экскурсия;
- партнер;
- есть долг;
- источник заявки;
- гость/пользователь.

### Карточка заявки

Показываем:

- номер заявки;
- текущий статус;
- данные клиента;
- выбранную экскурсию;
- дату;
- количество людей;
- комментарий;
- партнера;
- Telegram партнера;
- цену в рублях;
- цену в THB;
- комиссию;
- статус комиссии;
- историю статусов;
- заметку админа;
- технические данные Google Sheets.

### Действия

- изменить статус;
- переназначить партнера;
- добавить заметку админа;
- отметить отмену;
- отметить оказанной;
- пересчитать комиссию;
- открыть клиента;
- открыть экскурсию;
- открыть партнера.

### Опасные действия

Требуют подтверждения:

- отменить заявку;
- отметить заявку оказанной;
- изменить партнера после отправки заявки;
- изменить комиссию вручную.

## 3. Экскурсии

### Цель

Админ управляет каталогом экскурсий.

### Список экскурсий

Поля:

- название;
- статус;
- категория;
- цена от RUB;
- цена от THB;
- партнер;
- количество заявок;
- дата обновления;
- SEO-статус;
- фото есть/нет.

### Карточка экскурсии

Показываем:

- название;
- slug;
- категория;
- короткий эмоциональный текст;
- описание;
- маршрут;
- длительность;
- цена THB;
- цена RUB;
- курс;
- дата курса;
- предупреждение о курсе;
- что включено;
- что оплачивается отдельно;
- что взять с собой;
- ограничения;
- страховка;
- партнер;
- фото;
- отзывы;
- SEO title;
- SEO description;
- статус публикации.

### Действия

- создать экскурсию;
- изменить экскурсию;
- опубликовать;
- скрыть;
- добавить фото;
- заменить обложку;
- изменить SEO;
- открыть публичную страницу.

### Опасные действия

Требуют подтверждения:

- скрыть опубликованную экскурсию;
- изменить slug;
- изменить цену;
- изменить партнера;
- удалить фото.

Удаление карточки экскурсии не является обычным действием MVP.

Если когда-то понадобится полное удаление, оно должно быть только у админа и с отдельным подтверждением.

## 4. Пользователи

### Цель

Админ видит клиентов, историю заявок и баллы.

### Список пользователей

Поля:

- имя;
- телефон;
- Telegram;
- email;
- способ входа;
- количество заявок;
- баллы;
- уровень;
- дата регистрации.

### Карточка пользователя

Показываем:

- профиль;
- контакты;
- способ входа;
- история заявок;
- баллы;
- уровень;
- отзывы пользователя;
- заметка админа.

### Действия

- открыть заявку пользователя;
- добавить заметку;
- посмотреть историю баллов.

### Опасные действия

В MVP не делаем:

- удаление пользователя;
- ручное списание баллов;
- блокировку пользователя.

Эти действия можно добавить позже.

### Список отзывов

Поля:

- экскурсия;
- имя клиента;
- текст;
- фото есть/нет;
- статус;
- дата;
- связанная заявка.

### Карточка отзыва

Показываем:

- текст;
- фото;
- экскурсию;
- клиента;
- связанную заявку;
- статус;
- кто одобрил;
- дата публикации.

### Действия

- создать отзыв вручную;
- одобрить;
- скрыть;
- привязать к экскурсии;

- опубликовать отзыв;
- скрыть опубликованный отзыв.

- отфильтровать по партнеру;

## 10. Настройки

### Цель

Базовые настройки MVP.

### Показываем

- город по умолчанию: Phuket;
- страна по умолчанию: Thailand;
- язык по умолчанию: ru;
- валюта партнера: THB;
- публичная валюта: RUB;
- комиссия по умолчанию: 100 THB;
- текст предупреждения о курсе.

### Действия

- изменить текст предупреждения о курсе;
- изменить комиссию по умолчанию;
- изменить контакты проекта.

### Опасные действия

Требуют подтверждения:

- изменить комиссию по умолчанию;
- изменить валютные настройки.

## 11. Журнал действий

### Цель

Видеть важные действия в админке.

### Показываем

- кто сделал действие;
- когда сделал;
- что изменил;
- какую сущность изменил;
- старое и новое значение, если возможно.

### Важные события

- смена статуса заявки;
- изменение партнера;
- изменение цены;
- публикация экскурсии;
- скрытие экскурсии;
- публикация отзыва;
- изменение комиссии;
- отметка долга.

## Что не делаем в MVP админки

- много ролей админов;
- отдельную роль менеджера;
- полноценный партнерский кабинет;
- массовые рассылки;
- сложную CRM;
- удаление пользователей;
- самостоятельное удаление карточек партнерами;
- онлайн-оплаты.

## Задел на масштабирование

Админка должна быть спроектирована так, чтобы позже добавить:

- города;
- страны;
- языки;
- валюты;
- разные категории услуг;
- несколько админов;
- менеджеров;
- партнерские кабинеты;
- региональную аналитику.

В MVP значения по умолчанию:

- страна: Thailand;
- город: Phuket;
- язык: ru;
- валюта партнера: THB;
- публичная валюта: RUB.

## Definition of Done этапа

Этап карты экранов админки считается готовым, если описаны:

- разделы админки;
- поля в списках;
- поля в карточках;
- действия админа;
- опасные действия;
- что требует подтверждения;
- что не делаем в MVP;
- задел на масштабирование.

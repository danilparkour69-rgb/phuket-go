# Tech Stack

Статус: [~] recommended draft

## Рекомендация для MVP

Простыми словами:

- сайт и админка: Next.js;
- язык разработки: TypeScript;
- внешний вид: Tailwind CSS;
- временное хранение заявок: Google Sheets;
- целевая база: Postgres;
- авторизация: Firebase Authentication;
- Telegram-боты: отдельный backend-модуль;
- хостинг сайта: Yandex Cloud;
- хранение фото в MVP: Cloudinary.

## Почему так

- подходит для SEO-страниц экскурсий и статей;
- можно сделать сайт, админку и будущий Telegram Mini App на общей базе;
- удобно масштабировать на Postgres;
- можно не начинать с тяжелой мобильной разработки.

## MVP

В MVP делаем:

- первая версия запускается локально на компьютере;
- сайт;
- админку;
- форму заявки;
- Telegram-бот для партнеров;
- Telegram-бот для пользователей;
- Google Sheets как временное хранилище заявок;
- подготовку к Postgres;
- Firebase Authentication для бесплатных способов входа: Google и email;
- Telegram-вход без SMS там, где это технически возможно;
- Cloudinary для хранения фотографий экскурсий;
- Yandex Cloud как основной вариант хостинга.

## Порядок публикации

1. Локальный запуск на компьютере.
2. Проверка сайта, админки, заявок, ботов и Google Sheets.
3. Публикация на сервере.
4. Публикация/подключение остальных онлайн-площадок.

## Открытые вопросы

- выбрать точный сервис внутри Yandex Cloud для размещения сайта;
- определить технический процесс переноса данных из Google Sheets в Postgres.

## Принятые решения

- сервис авторизации: Firebase Authentication;
- вход по телефону через SMS в MVP не используем, потому что нужен бесплатный способ;
- хранилище фото для MVP: Cloudinary;
- хостинг: Yandex Cloud, не Yandex Disk;
- перенос в Postgres: после внесения и подготовки данных в Google Sheets.

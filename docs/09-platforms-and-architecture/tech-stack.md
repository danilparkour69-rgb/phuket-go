# Tech Stack

Статус: [x] done for planning

## Рекомендация для MVP после выбора шаблона

Используем шаблон `vibe`.

Простыми словами:

- публичный сайт: `website`, Astro;
- личный кабинет и админка: `webapp`, React + Vite;
- backend/API: `backend`, Bun + Hono;
- общие схемы данных: `packages/contracts`, Zod + TypeScript;
- язык разработки: TypeScript;
- внешний вид: Tailwind CSS и компоненты шаблона;
- временное хранение заявок: Google Sheets;
- целевая база: Postgres;
- авторизация: Firebase Authentication или встроенная авторизация шаблона после технической проверки;
- Telegram-боты: backend-модуль;
- хостинг после локальной проверки: Yandex Cloud;
- хранение фото в MVP: Cloudinary.

## Почему так

- `website` подходит для SEO-страниц экскурсий и статей;
- `webapp` подходит для личного кабинета и админки, потому что эти страницы не должны находиться через поиск;
- `backend` дает единую логику для сайта, админки, Telegram-ботов и будущих приложений;
- `packages/contracts` помогает держать одинаковые правила данных во всех частях проекта;
- можно начать локально и позже перенести на сервер;
- можно не начинать с тяжелой мобильной разработки.

## MVP

В MVP делаем:

- первая версия запускается локально на компьютере;
- публичный сайт в `website`;
- личный кабинет и админку в `webapp`;
- backend/API в `backend`;
- форму заявки;
- Telegram-бот для партнеров;
- Telegram-бот для пользователей;
- Google Sheets как временное хранилище заявок;
- подготовку к Postgres;
- бесплатные способы входа: Telegram, Google, email;
- Telegram-вход без SMS там, где это технически возможно;
- Cloudinary для хранения фотографий экскурсий;
- Yandex Cloud как основной вариант хостинга.

## Порядок публикации

1. Локальный запуск на компьютере.
2. Проверка сайта, админки, заявок, ботов и Google Sheets.
3. Публикация на сервере.
4. Публикация/подключение остальных онлайн-площадок.

## Открытые вопросы

- выбрать точный сервис внутри Yandex Cloud для размещения сайта;
- определить технический процесс переноса данных из Google Sheets в Postgres.
- финально выбрать авторизацию после проверки возможностей шаблона: Firebase Authentication или встроенная авторизация `vibe`.

## Принятые решения

- вход по телефону через SMS в MVP не используем, потому что нужен бесплатный способ;
- хранилище фото для MVP: Cloudinary;
- хостинг: Yandex Cloud, не Yandex Disk;
- перенос в Postgres: после внесения и подготовки данных в Google Sheets.
- шаблон проекта: `vibe`;
- `vibe` используем как стартовый шаблон, а не как точную архитектуру Phuket Go.

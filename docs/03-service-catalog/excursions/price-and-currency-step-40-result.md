## Цель

Зафиксировать, как Phuket Go работает с ценами в MVP:

- что хранится внутри базы;
- что показывается пользователю;
- какие цены уже исправлены;
- какие цены нельзя публиковать без уточнения.

## Главное решение

Внутри базы и для партнера цена хранится в THB.

Пользователю в русскоязычном MVP показываем цену в рублях.

Формат на сайте:

```text
от 8 500 ₽
```

Под ценой мелким текстом:

```text
Цена рассчитана по текущему курсу. Из-за изменения курса рубля итоговая сумма может отличаться.
```

## Что не делаем сейчас

На этапе планирования не фиксируем:

- точный курс THB/RUB;
- источник курса;
- частоту обновления курса;
- округление цены в рублях;
- автоматический пересчет.

Это будет решаться перед публикацией.

## Правило цены "от"

Для публичной карточки берем минимальную взрослую цену основной экскурсии.

Не используем как цену "от":

- детский тариф;
- входной билет;
- национальный парк;
- депозит;
- трансферную доплату;
- цену отдельной опции;
- техническую цену из источника;
- подозрительно низкую цену без подтверждения.

## Решение для сайта

Если цена не подтверждена, в публичной карточке нельзя показывать подозрительную цену.

Варианты до уточнения:

- не публиковать карточку;
- показывать `Цена уточняется`;
- показывать карточку только в подборке без цены;
- отправлять такую карточку в ручную заявку.

Для MVP лучше:

- популярные экскурсии публиковать только с проверенной ценой;
- карточки со спорной ценой оставить в базе, но не выводить как готовые к продаже.

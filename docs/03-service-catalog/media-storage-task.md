# Media Storage Task

Статус: [~] draft

## Задача

Определить, где хранить фотографии экскурсий.

## Что нужно решить

- где хранить фото;
- как загружать фото в админке;
- сколько фото можно добавить к одной экскурсии;
- какое фото является обложкой;
- нужны ли разные размеры фото для сайта и Telegram Mini App;
- как переносить фото при переходе на полноценную архитектуру.

## MVP

В первом MVP достаточно фотогалереи.

Видео в первом MVP не нужно.

## Решение для MVP

Фотографии экскурсий сначала храним локально внутри проектной папки.

Локальная папка для подготовки медиа:

- `docs/03-service-catalog/media/excursions/`

Причина: сейчас проект находится на этапе подготовки базы и контента. До публикации не выбираем внешний сервис, чтобы не усложнять работу и не привязываться к платформе раньше времени.

Для каждой экскурсии позже нужно создать отдельную папку:

- `docs/03-service-catalog/media/excursions/001-slon-mantra-spa-samet-nangshe/`
- `docs/03-service-catalog/media/excursions/018-phi-phi/`

Правило хранения:

- `original/` - исходные фото;
- `final/carousel/` - фото для верхней карусели;
- `final/content/` - будущие фото внутри текста;
- `media-manifest.json` - список фото, источники, типы и статусы.

## Нейросетевые изображения

Позже можно использовать нейросети для:

- улучшения качества фото;
- расширения кадра;
- удаления технических дефектов;
- генерации атмосферных изображений для статей, обложек или рекламных материалов.

Важно:

- реальные фото экскурсии храним отдельно от AI-изображений;
- AI-изображение нельзя выдавать за реальное фото конкретной экскурсии, лодки, гида, отеля или партнера;
- если изображение создано нейросетью с нуля, в медиабазе ставим тип `ai_generated`;
- если реальное фото улучшено нейросетью, ставим тип `ai_enhanced`;
- для карточек экскурсий предпочтительнее реальные фото, потому что пользователь выбирает конкретную поездку;
- AI-изображения можно использовать как иллюстрации эмоции, если они не обещают конкретный объект или точную программу.

Цель визуала: выглядеть красиво, реалистично и живописно, но не вводить пользователя в заблуждение.

## Future

Варианты:

- Yandex Object Storage, если проект будет размещаться в Yandex Cloud;

- другое S3-совместимое хранилище.

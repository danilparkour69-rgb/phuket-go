## Принцип

Для каждой крупной категории будет своя база данных.

Примеры:

- база экскурсий;
- база байков;
- база недвижимости;
- база документов/виз;
- база других категорий.

## Задача MVP

Собрать базу экскурсий для MVP.

Берем все экскурсии из источника Marusya Travel:

- Telegram-пост: https://t.me/marusyatravel/2206
- список и цены: `initial-excursions-source-marusya.md`
- подробные ссылки: `initial-excursions-links-marusya.md`

## Что нужно собрать

- название экскурсии;
- направление;
- описание;
- варианты внутри экскурсии;
- цена "от";
- цена каждого варианта;
- длительность, рассчитанная по расписанию дня, если в источнике есть начало и финал;
- источник расчета длительности и пометка, когда время зависит от трансфера или выбранного варианта;
- маршрут;
- что включено;
- что взять с собой;
- страховка;
- условия отмены;
- фото;
- партнер/исполнитель для внутренней админки;
- комиссия/условия с партнером.

Подробная структура колонок описана в `excursions-database-structure.md`.

- `initial-excursions-source-marusya.md`

Названия экскурсий в посте являются гиперссылками на подробные Telegra.ph-страницы с описаниями, фото и ценами.

Важно: это источник для структуры и списка, не готовые тексты для Phuket Go.

## Статус

В работе: первый источник получен, в MVP-базу берем все экскурсии из списка.

Правило по длительности: не задаем партнеру отдельный вопрос "сколько длится экскурсия", если в подробной ссылке уже есть расписание. Сначала считаем сами по первому и последнему времени программы. Уточняем только неполные случаи: нет финального времени, есть только время начала, или длительность зависит от выбранного рейса/сеанса/аренды.

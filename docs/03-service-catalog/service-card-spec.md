# Service Card Spec

Статус: [~] draft

## Основные поля

- название;
- техническое название для пользователя, базы, SEO и админки;
- продающее название для карточки;
- описание;
- категория;
- направление экскурсии;
- маршрут;
- длительность;
- расписание/доступные даты;
- что взять с собой;
- что предоставляется;
- страховка;
- условия отмены;
- партнер/исполнитель, только для админки;
- статус публикации.

Перед публикацией все публичные названия экскурсии должны быть полностью на русском языке: `title`, `selling_title`, H1, SEO title, карточка каталога и любые видимые подписи. Латиница допустима только для технических полей вроде `slug`, ID, имен файлов и внутренних ключей.

## Цена и условия

- цена "от" на карточке показывается в рублях;
- под ценой мелким текстом показывается предупреждение: "Цена рассчитана по текущему курсу. Из-за изменения курса рубля итоговая сумма может отличаться.";
- длительность экскурсии показывается на публичной карточке рядом с ценой и ключевыми маркерами;
- исходная цена партнера хранится в THB;
- в будущем варианты внутри экскурсии показываются как список товаров/вариантов, например: стандарт, VIP, с ночевкой;
- в первом MVP пользователь оставляет одну общую заявку, без выбора конкретного варианта;
- публичная валюта для русскоязычного MVP: RUB;
- внутренняя валюта партнера: THB;
- что включено в цену;
- что оплачивается отдельно;
- ограничения по возрасту/количеству людей, если есть.

## Баллы для авторизованных пользователей

- в карточке можно показывать, сколько баллов пользователь получит после посещения экскурсии;
- баллы видит только авторизованный пользователь;
- гость не видит персональный блок баллов, чтобы карточка оставалась простой;
- баллы начисляются не за заявку, а после подтвержденного посещения;
- точная формула баллов будет рассчитана позже;
- более дорогие и сложные экскурсии должны давать больше баллов.

## Кнопки

- основная кнопка: "Оставить заявку";
- будущий вариант CTA: `Бесплатное бронирование` / `Забронировать бесплатно`, если условия подтверждены и текст не создает ложное ощущение финального бронирования без ответа партнера;
- пользователь заполняет форму и нажимает "Отправить заявку";
- отдельную кнопку "Написать в Telegram" в MVP не делаем.
- кнопка "Оставить заявку" должна органично повторяться в разных местах карточки при прокрутке сверху вниз.
- в будущем можно добавить сердечко `Хочу посетить` / `В избранное` для пользователя, который заинтересовался, но еще не готов выбрать дату.
- в будущем можно добавить кнопку `Задать вопрос`: общие вопросы обрабатывает AI-ассистент, вопросы по конкретной экскурсии, цене, дате, оплате и условиям передаются менеджеру.

## Фильтры MVP

- категория/направление экскурсии;
- цена.

## SEO-поля

- URL slug;
- title;
- description;
- h1;
- краткое SEO-описание;
- ключевые направления/локации.

## Медиа

- фото;
- обложка карточки;
- галерея;
- верхняя свайпаемая карусель на 3-5 сильных изображений;
- контекстные изображения внутри текста: паром, завтрак, снаряжение, остановки маршрута и другие детали, о которых говорится в конкретном блоке.
- видео в первом MVP не нужно.

## Фото

Нужно отдельно решить, где будут храниться фотографии экскурсий.

Статус вопроса: не решено.

## Отзывы

- отзывы показываются в конце карточки экскурсии;
- в MVP отзывы добавляет администратор вручную;
- формат: текстовый отзыв;
- рейтинг звездами не нужен.

## Модерация

- карточки создает и редактирует администратор;
- отзывы в MVP добавляет администратор вручную;
- маркеры доверия: проверенный партнер, русскоязычный гид, страховка, отзывы, условия отмены.
- партнер/исполнитель не показывается пользователю.

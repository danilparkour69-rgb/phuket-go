# Service Card Spec

Статус: [~] draft

## Основные поля

- название;
- описание;
- категория;
- направление экскурсии;
- маршрут;
- длительность;
- расписание/доступные даты;
- что взять с собой;
- что предоставляется;
- страховка;
- условия отмены;
- партнер/исполнитель, только для админки;
- статус публикации.

## Цена и условия

- цена "от" на карточке показывается в рублях;
- под ценой мелким текстом показывается предупреждение: "Цена рассчитана по текущему курсу. Из-за изменения курса рубля итоговая сумма может отличаться.";
- исходная цена партнера хранится в THB;
- в будущем варианты внутри экскурсии показываются как список товаров/вариантов, например: стандарт, VIP, с ночевкой;
- в первом MVP пользователь оставляет одну общую заявку, без выбора конкретного варианта;
- публичная валюта для русскоязычного MVP: RUB;
- внутренняя валюта партнера: THB;
- что включено в цену;
- что оплачивается отдельно;
- ограничения по возрасту/количеству людей, если есть.

## Кнопки

- основная кнопка: "Оставить заявку";
- пользователь заполняет форму и нажимает "Отправить заявку";
- отдельную кнопку "Написать в Telegram" в MVP не делаем.
- кнопка "Оставить заявку" должна органично повторяться в разных местах карточки при прокрутке сверху вниз.

## Фильтры MVP

- категория/направление экскурсии;
- цена.

## SEO-поля

- URL slug;
- title;
- description;
- h1;
- краткое SEO-описание;
- ключевые направления/локации.

## Медиа

- фото;
- обложка карточки;
- галерея.
- видео в первом MVP не нужно.

## Фото

Нужно отдельно решить, где будут храниться фотографии экскурсий.

Статус вопроса: не решено.

## Отзывы

- отзывы показываются в конце карточки экскурсии;
- в MVP отзывы добавляет администратор вручную;
- формат: текстовый отзыв;
- рейтинг звездами не нужен.

## Модерация

- карточки создает и редактирует администратор;
- отзывы в MVP добавляет администратор вручную;
- маркеры доверия: проверенный партнер, русскоязычный гид, страховка, отзывы, условия отмены.
- партнер/исполнитель не показывается пользователю.

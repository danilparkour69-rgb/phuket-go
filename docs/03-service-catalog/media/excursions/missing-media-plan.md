# Missing Media Plan

Статус: [~] рабочий план

Дата: 2026-06-28

Важно: Pexels больше не используем для публикационной базы. Временные Pexels-фото, которые были загружены на раннем этапе, удалены из проекта. Недостающие фото закрываем партнерскими материалами или новыми изображениями по фотосценарию.

## Цель

Понять, какие фото нужны для первого пакета экскурсий и как их добирать без ручного хаоса.

Общий инвентарь по всем 54 экскурсиям ведется отдельно:

- `all-excursions-media-inventory.md`

## Главное правило

Каждое фото должно иметь понятный источник в `source.txt`.

Тип фото:

- `real` - реальное фото из источника или от партнера;
- `ai_enhanced` - реальное фото, улучшенное через нейросеть;
- `ai_generated` - полностью сгенерированное фото.

AI-фото нельзя выдавать за реальное фото конкретной экскурсии. Внутри проекта тип должен быть указан явно.

## Проверка источников

Исходные страницы проверены. По частичным карточкам дополнительных фото в источниках не хватает.

Пользователь уточнил: ранее переданная строка была связана с доступом к Pexels API, а не с папкой фото.

Рабочий источник для временного добора фото:

`https://www.pexels.com/ru-ru/api/key/`

Ключ Pexels API нельзя записывать в документы и нельзя коммитить в проект. Если он понадобится при разработке, хранить только в локальном `.env`.

## Временное решение

Пока фото из базы фото не получены отдельно, пробуем брать недостающие фото через Pexels API.

Если фото подходит по смыслу и качеству, его можно использовать как временный источник для добора галерей. В `source.txt` у каждого файла нужно указать:

- `source_owner: Pexels`;
- `usage_permission: Pexels API/source, final license check needed`;
- `image_type: real`, если фото реальные;
- `notes: temporary Pexels source for MVP media completion`.

## Приоритет

1. Получить реальные фото от партнера.
2. Если партнер не дает фото - использовать разрешенные фото из открытых/покупных источников.
3. Если фото слабого качества - делать `ai_enhanced`.
4. Если нет подходящих фото - делать `ai_generated`, но хранить это как отдельный тип.

AI-генерацию запускать параллельно через субагента. Промпты должны опираться на публичный текст конкретной экскурсии и на принцип: продаем эмоцию, мечту и воспоминание, а не просто экскурсию.

## Что просить у партнера

По каждой экскурсии лучше получить:

- 1 горизонтальное фото для обложки;
- 3-5 фото для галереи;
- фото без водяных знаков и чужих логотипов;
- разрешение на сайт, рекламу и соцсети.

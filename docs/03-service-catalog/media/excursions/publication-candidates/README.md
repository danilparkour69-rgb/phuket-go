# Publication Candidates

Дата: 2026-06-28

## Правило

Для публикационной базы по экскурсиям берем все доступные фото по направлению, кроме:

- фото из отзывов.

Сначала сохраняем полный набор доступных фото, потом отдельно выбираем:

- обложку;
- карусель: только горизонтальные фото;
- фото внутри текста;
- фото, которые нужно улучшить или заменить.

Перед публикацией карусель проверяем отдельно: на верхних превью не должно быть крупных узнаваемых лиц людей. Если такие фото есть, отмечаем их в списке и готовим замену или AI-генерацию.

## Порядок перед публикацией направления

1. Сначала готовим список фото на удаление.
2. Удаляем неподходящие фото после подтверждения.
3. Сразу после этого готовим список фото, которые нужно перегенерировать и загрузить.
4. Для перегенерации указываем, какую роль должно закрывать фото: обложка, карусель, маршрут, транспорт, активность, питание, безопасность или эмоция.

## Общая страница отбора

Для всех направлений создана общая страница:

- `../all-publication-photo-review.html`
- `../publication-photo-action-list.md`
- `../missing-carousel-photo-review.html`

В ней фото разделены по направлениям. Для каждого фото можно отметить:

- `удалить`;
- `перегенерировать/залить`.

Справа формируются два отдельных списка, уже сгруппированные по направлениям.

`publication-photo-action-list.md` показывает по каждому направлению:

- сколько фото осталось после удаления;
- сколько горизонтальных фото подходит для верхней карусели;
- где нужно добрать или перегенерировать фото;
- где аналог экскурсии неточный и требует проверки.

В этом файле также есть блок `Что именно добрать или перегенерировать`: там кратко описано, какие кадры нужны для направлений, где не хватает фото.

Подробные брифы для генерации или запроса новых фото:

- `../image-regeneration-briefs.md`

Для быстрой проверки только направлений, где не хватает горизонтальных фото для карусели, использовать:

- `../missing-carousel-photo-review.html`

## Главное решение

На этапе подготовки MVP не зависаем на идеальном подборе фото.

Для сайта берем доступные фото, которые:

- лежат в папке конкретного направления;
- соответствуют карточке экскурсии;
- перечислены в `media-manifest.json`.

Красоту, замену слабых фото, проверку лиц и ручную чистку переносим на этап перед публикацией.

## Что использовать будущему сайту

Для каждой экскурсии сайт должен читать:

```text
docs/03-service-catalog/media/excursions/{direction}/media-manifest.json
```

Карусель брать из:

```text
docs/03-service-catalog/media/excursions/{direction}/final/carousel/
```

Фото внутри текста позже брать из:

```text
docs/03-service-catalog/media/excursions/{direction}/final/content/
```

Пока `final/content` может быть пустой.

## Что не использовать напрямую

Эти папки нужны для подготовки, но сайт не должен брать из них фото напрямую:

```text
original/
original/photo-base/
carousel-candidates/
photo-base/
```

Почему:

- там могут быть исходники;
- там могут быть дубли;
- там могут быть слабые фото;
- там могут быть фото, которые позже заменим.

## Как выбирать фото для карусели

Берем все фото из `media-manifest.json`, где:

- `role`: `carousel`;
- `file` ведет в `final/carousel`;
- `status` не равен `rejected`.

Сайт показывает все подходящие фото, которые отобраны для карусели в `media-manifest.json`.

## Какие статусы сейчас допустимы

| Статус | Что делать сейчас |
| --- | --- |
| `ready_for_manual_review` | Можно использовать как временное фото |
| `needs_replacement_later` | Можно использовать временно, если другого фото нет |

Перед публикацией желательно перевести финальные фото в статус:

```text
approved
```

## Цель

Расписать, какие фото нужны для проблемных и приоритетных экскурсий:

- первые 4-5 фото в карусели;
- дополнительные фото по тексту страницы;
- что запросить у партнера;
- что можно брать из выбранных источников;
- что можно генерировать с нуля.

Важно: фото с Tripster и Sputnik8 используем как референсы по сценам. Финальные изображения для Phuket Go готовим отдельно.

Отдельный пример по Пхи-Пхи:

- `phi-phi-competitor-photo-reference.md`

## Приоритет 1

### 010 Паром на Пхи-Пхи

Проблема: весь набор фото низкого качества и с дублями.

Карусель:

1. Паром или крупный теплоход на фоне моря.
2. Пирс и посадка на паром.
3. Вид с палубы на море.
4. Прибытие к Пхи-Пхи или вид острова с воды.
5. Люди на палубе/в салоне без ощущения хаоса.

Фото по тексту:

- билет/посадка;
- салон или палуба;
- багаж/организация поездки;
- море по пути;
- прибытие на остров.

Запросить у партнера:

- фото реального парома;
- фото пирса;
- фото посадки;
- фото салона/палубы.

Можно из стока:

- нейтральное море;
- вид острова с воды.

AI:

- можно генерировать только общий вид парома/моря без логотипов и без обещания конкретного судна.

### 013 Симиланы Премиум

3. Смотровая точка или узнаваемый вид.
4. Снорклинг/маска/вода.
5. Спокойный премиальный отдых без толпы.

Фото по тексту:

- посадка на катер;
- пляж;
- вода и снорклинг;
- питание, если включено;
- группа/гид, если подтверждено.

Запросить у партнера:

- пляж без перегруза людьми;
- фото смотровой;
- фото питания/организации.

Можно из стока:

AI:

- можно сделать атмосферный белый пляж и воду, но с пометкой `ai_generated`.

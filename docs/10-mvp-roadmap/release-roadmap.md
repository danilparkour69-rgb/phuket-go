# Release Roadmap

Статус: [~] draft

## Этап 1. Подготовка данных и структуры

- подготовить структуру экскурсий;
- подготовить структуру Google Sheets;
- подготовить базу всех экскурсий из первого источника Marusya Travel;
- подготовить список первых статей;
- подготовить требования к фото.

Статус сейчас:

- база 54 экскурсий собрана;
- фото не блокируют дальнейшую работу;
- для MVP берем доступные фото из папки конкретного направления, если они соответствуют карточке экскурсии;
- финальную ручную чистку фото переносим на этап перед публикацией;
- критичных блокеров локальной разработки нет.

## Этап 2. Сайт и SEO-страницы

- главная страница;
- каталог экскурсий;
- страница каждой экскурсии;
- статьи;
- FAQ;
- контакты;
- условия бронирования.

Важно:

- публичные страницы делаются в `website`;
- каждая экскурсия должна иметь отдельную SEO-страницу;
- фото страницы экскурсии берутся через `media-manifest.json` и `final/carousel`.

## Этап 3. Админка

- управление экскурсиями;
- управление заявками;
- управление пользователями;
- управление отзывами;
- управление статьями;
- просмотр комиссий;
- просмотр аналитики.

## Этап 4. Заявки и Google Sheets

- форма заявки;
- отправка заявки;
- запись заявки в Google Sheets;
- отображение заявки в админке;
- история статусов.

## Этап 5. Telegram-боты

- бот для партнеров;
- бот для пользователей;
- статусы заявок;
- уведомления.

## Этап 6. Авторизация и личный кабинет

- вход пользователя;
- история заявок;
- баллы;
- первый уровень.

## Этап 7. Баллы и уровни

- 100 баллов за регистрацию;
- баллы за экскурсию;
- баллы за отзыв;
- баллы за отзыв с фото;
- баллы за реферала.

## Этап 8. Аналитика и комиссии

- количество заявок;
- заявки по экскурсиям;
- заявки по партнерам;
- оказанные заявки;
- сумма комиссии;
- пометка долга.

## Этап 9. Тестирование и запуск

- сначала запустить первую версию на локальном компьютере;
- проверить локально весь основной функционал;
- проверить путь пользователя;
- проверить путь партнера;
- проверить админку;
- проверить Google Sheets;
- проверить уведомления;
- проверить SEO-страницы;
- после локальной проверки публиковать на сервере и онлайн-площадках.

Перед публикацией отдельно проверить:

- актуальные цены;
- Telegram-получателей заявок;
- домен и хостинг;
- боевые Telegram-боты;
- Google Sheets;
- юридические тексты;
- финальную ручную чистку фото.

## Definition of Done

- [x] Дорожная карта отражает текущее решение по фото.
- [x] Дорожная карта отделяет локальную разработку от публикации.
- [x] Зафиксировано, что фото не блокируют следующий этап.

# Release Roadmap

Статус: [~] draft

## Текущий release gate

Дата обновления: 2026-07-02.

Публикацию пока не начинаем. У проекта уже есть направление к домену/URL, но deployment workflow заблокирован до локального 10/10.

Перед любым деплоем нужно сначала закрыть:

- финальную верстку и визуальный аудит публичного сайта;
- полный локальный путь заявки: website -> backend -> Telegram admin/partner -> Google Sheets -> admin queue;
- решение по оставшимся MVP-поверхностям: что закрываем сейчас, что явно откладываем;
- актуальные docs/task map;
- чистый git status, зафиксированный commit и зеленые проверки.

## Этап 1. Подготовка данных и структуры

- подготовить структуру экскурсий;
- подготовить структуру Google Sheets;
- подготовить базу всех экскурсий из первого источника Marusya Travel;
- подготовить список первых статей;
- подготовить требования к фото.

Статус сейчас:

- база 54 экскурсий собрана;
- фото не блокируют дальнейшую работу;
- для MVP берем доступные фото из папки конкретного направления, если они соответствуют карточке экскурсии;
- финальную ручную чистку фото переносим на этап перед публикацией;
- критичных блокеров локальной разработки нет.

## Этап 2. Сайт и SEO-страницы

- главная страница;
- каталог экскурсий;
- страница каждой экскурсии;
- статьи;
- FAQ;
- контакты;
- условия бронирования.

Важно:

- публичные страницы делаются в `website`;
- каждая экскурсия должна иметь отдельную SEO-страницу;
- фото страницы экскурсии берутся через `media-manifest.json` и `final/carousel`.

Статус сейчас:

- страницы сайта уже собираются локально;
- `website` использует локальный media sync и не должен зависеть от localhost media URL в production build;
- главная и мобильная страница экскурсии получили первый polish;
- этап не закрыт: требуется финальный visual/content pass всех публичных страниц до 10/10.

## Этап 3. Админка

- управление экскурсиями;
- управление заявками;
- управление пользователями;
- управление отзывами;
- управление статьями;
- просмотр комиссий;
- просмотр аналитики.

## Этап 4. Заявки и Google Sheets

- форма заявки;
- отправка заявки;
- запись заявки в Google Sheets;
- отображение заявки в админке;
- история статусов.

## Этап 5. Telegram-боты

- бот для партнеров;
- бот для пользователей;
- статусы заявок;
- уведомления.

## Этап 6. Авторизация и личный кабинет

- вход пользователя;
- история заявок;
- баллы;
- первый уровень.

## Этап 7. Баллы и уровни

- 100 баллов за регистрацию;
- баллы за экскурсию;
- баллы за отзыв;
- баллы за отзыв с фото;
- баллы за реферала.

## Этап 8. Аналитика и комиссии

- количество заявок;
- заявки по экскурсиям;
- заявки по партнерам;
- оказанные заявки;
- сумма комиссии;
- пометка долга.

## Этап 9. Тестирование и запуск

- сначала запустить первую версию на локальном компьютере;
- проверить локально весь основной функционал;
- проверить путь пользователя;
- проверить путь партнера;
- проверить админку;
- проверить Google Sheets;
- проверить уведомления;
- проверить SEO-страницы;
- после локальной проверки и отдельного решения владельца проекта публиковать на сервере и онлайн-площадках.

До выполнения этих пунктов не создавать production cloud resources, не загружать frontend build output в buckets/static hosting и не запускать backend image push.

Перед публикацией отдельно проверить:

- актуальные цены;
- Telegram-получателей заявок;
- домен и хостинг;
- боевые Telegram-боты;
- Telegram polish: текущие кнопки и последовательность уже реализованы; стикеры не добавляем, пока нет утвержденных Telegram sticker assets;
- Google Sheets;
- юридические тексты;
- финальную ручную чистку фото.

## Будущий этап. Мобильное приложение

Не входит в MVP.

Когда дойдем до Android/iPhone:

- подготовить мобильное приложение на отдельном этапе;
- сделать ASO для App Store и Google Play;
- подготовить название, subtitle/short description, описание, ключевые слова, скриншоты, preview/video и локализации;
- проверить конкурентов в сторах перед публикацией.

## Definition of Done

- [x] Дорожная карта отражает текущее решение по фото.
- [x] Дорожная карта отделяет локальную разработку от публикации.
- [x] Зафиксировано, что фото не блокируют следующий этап.
- [x] ASO для будущего приложения зафиксировано как задача не-MVP этапа.

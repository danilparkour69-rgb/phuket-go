# Короткие тексты карточек для главной и каталога

Статус: [x] draft для будущей разработки

Кодинг проекта пока не начинаем.

Документ содержит короткие эмоциональные тексты для карточек приоритетных экскурсий. Тексты нужны для главной страницы, подборок и каталога.

Главный принцип: карточка продает не маршрут, а желание оказаться в этом дне.

## Названия в карточке

В карточке разделяем два названия:

- техническое название - нужно для базы, SEO, админки, поиска и понимания пользователем, куда он едет;
- продающее название - нужно для человека на сайте.
- короткий текст - раскрывает обещание без длинной программы.

Пример:

- техническое: `052. Яхты, лодки, катера`;
- продающее: `Ваш день в баунти-раю на воде`;
- короткий текст: `Море, маршрут и темп отдыха, который можно собрать под свое настроение.`

Техническое название не меняем без причины. Продающее название можно менять, усиливать и тестировать.

На сайте техническое название тоже показываем человеку. Оно отвечает на вопрос “куда я отправляюсь”.

Подробная проверка названий сохранена в:

- `../03-service-catalog/excursions/title-check-step-44-result.md`

Главное правило: продающее название должно начинаться с желания, картинки или эмоции. Сначала человек должен захотеть туда попасть, потом уже понять формат.

Подробная проверка эмоционального текста сохранена в:

- `../03-service-catalog/excursions/emotional-copy-step-45-result.md`

Пример структуры карточки:

1. Техническое направление: `Пхи-Пхи`.
2. Продающее название: `Бирюзовые бухты Пхи-Пхи`.
3. Короткий эмоциональный текст.
4. Цена.
5. Для авторизованных пользователей: сколько баллов можно получить после посещения.
6. Кнопка.

## Будущие тесты названий

Позже можно делать 2-3 варианта продающего названия для одной экскурсии и смотреть, где выше переходы и заявки.

Пример для Пхи-Пхи:

- `Бирюзовые бухты Пхи-Пхи`;
- `Острова, ради которых летят на Пхукет`;
- `Пхи-Пхи: день среди скал и бирюзовой воды`.

Пример для яхт:

- `Ваш день в баунти-раю`;
- `Море, яхта и день только для вас`;
- `Личный морской день как из отпуска мечты`.

Тестируем только продающее название и первую строку. Техническое название, slug, SEO-адрес и внутренняя карточка остаются стабильными.

## Правило первой фразы

Первая фраза должна начинаться с самой сильной картинки или эмоции.

Сначала:

- баунти-рай;
- бирюзовая вода;
- скалы как в фильме;
- закат на палубе;
- тишина озера;
- энергия тайского бокса;
- подводный мир;
- белый песок;
- джунгли;
- вид, ради которого стоит проснуться.

Потом уже объясняем формат.

Не начинать с:

- `формат для тех, кто...`;
- `маршрут для тех, кто...`;
- `день для тех, кто...`;
- `способ провести...`;
- `несколько морских настроений...`.

## Правило карточки

Карточка должна быть короткой:

- техническое название;
- продающее название;
- 1 эмоциональная фраза;
- цена от в батах;
- примерная цена в рублях для русской версии;
- баллы за посещение для авторизованных пользователей;
- кнопка `Оставить заявку`.

Не пишем в карточке:

- длинную программу;
- технические условия;
- имя партнера;
- неподтвержденные обещания.

Короткая фраза в карточке должна рисовать один момент, а не пересказывать программу.

## Баллы в карточке

Баллы показываем только авторизованному пользователю.

Гость видит:

- техническое название;
- продающее название;
- короткий текст;
- цену;
- кнопку заявки.

Авторизованный пользователь дополнительно видит:

- `+N баллов после посещения`;
- свой уровень или прогресс до следующего уровня, если это уместно в интерфейсе.

Баллы начисляются не за отправку заявки, а после подтвержденного посещения экскурсии.

Пример:

- `Пхи-Пхи`
- `Бирюзовые бухты Пхи-Пхи`
- `Бирюзовая вода, скалы и пляжи...`
- `+180 баллов после поездки`

Точную шкалу баллов рассчитываем позже.

# Mobile Plan

Статус: отложено после MVP Phuket Go

## Роль mobile в проекте

Мобильное приложение не входит в текущий MVP Phuket Go.

Текущий MVP запускается через:

- `website` для публичного каталога и SEO-страниц;
- `webapp` для личного кабинета и админки;
- `backend` для API, заявок, авторизации и Telegram.

## Что важно заранее

- mobile живет отдельно от `website` и `webapp`;
- mobile использует общий `backend` и `packages/contracts`;
- mobile не должен дублировать бизнес-логику backend;
- мобильная ветка шаблона берется позже из `mobile` branch.

## Что будем делать позже

Когда Phuket Go MVP на web будет стабилен, отдельным этапом:

1. переключаемся на `mobile` branch шаблона;
2. уточняем пользовательские mobile-сценарии;
3. определяем MVP мобильного клиента;
4. подключаем Expo, dev build и Maestro;
5. отдельно принимаем решения по платным mobile-ресурсам и сторам.
